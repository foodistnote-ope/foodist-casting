# 機能説明書 (Functional Specification)
**対象システム**: Foodist Casting (社内管理用 候補者選定ツール)

## 1. システムの目的
本システムは、アイランド株式会社の担当者が、仕事案件に対して最適な「フーディスト」を素早く選定し、キャスティング業務を円滑に進めるための社内管理システムです。マネタイズは行わず、純粋に社内の業務効率化（検索の手間削減、情報の属人化防止）を目的とします。

## 2. ユーザー権限
本システムを利用するユーザーは以下の2種類を想定します。

1. **システム管理者 (アイランド社内担当者)**
   - フーディスト情報の検索・閲覧・登録・編集・削除
   - 案件（プロジェクト）の作成、候補者リストの作成とステータス管理
   - 情報登録フォームのURL発行
2. **フーディスト (外部登録者)**
   - 担当者から送られたフォームURLからの情報入力・送信
   - ※システムへのログイン機能や他の候補者の閲覧権限は持たない。

## 3. 主要な画面と機能要件

### 3.1 ダッシュボード（管理者トップ画面）
* **機能**:
  * 登録されているフーディストの総数、最近登録されたフーディストの一覧表示。
  * 進行中のキャスティング案件（候補者選定中）のリスト表示。

### 3.2 フーディスト データベース機能
* **フーディスト一覧・検索画面**
  * **機能**: 登録済みの全フーディストを一覧表示。
  * **検索フィルター**: 
    * 属性（料理家、フードスタイリスト、管理栄養士など）
    * SNSフォロワー数（〇〇人以上など）
    * 得意ジャンル（和食、スイーツ、動画制作可、顔出し可など）
    * フリーワード検索（自己PRや経歴などから全文検索）
* **フーディスト詳細画面**
  * **機能**: 各フーディストのプロフィール詳細、SNSリンク、過去の起用履歴、社内メンバー専用の評価メモを表示。
* **手動登録・編集画面**
  * **機能**: 担当者がSNS等で見つけた魅力的な候補者を新規登録、または既存のデータを手動で修正・更新する。

### 3.3 候補者選定（キャスティング）機能
* **案件管理画面**
  * **機能**: 「〇〇食品 新商品PRアンバサダー」などの案件（プロジェクト）を作成し、案件ごとに必要な条件をメモする。
* **リストアップ（候補者追加）機能**
  * **機能**: データベースから検索したフーディストを、特定の案件の「候補者リスト」に追加する。
  * **ステータス管理**: リストアップされた各フーディストに対し、「検討中」「打診中」「決定」「辞退」などのステータスを付与して進捗を管理する。

### 3.4 フォーム連携機能
* **情報登録フォーム画面 (外部向け公開ページ)**
  * **機能**: フーディスト自身がプロフィール、得意分野、SNSアカウントURL、アピールポイントなどを入力するアンケート形式のWeb画面。
* **回答の取り込み**
  * **機能**: フーディストがフォームを送信すると、自動的にデータベースへ新規登録（または「未確認一覧」に入り、管理者が承認後に本登録）される機能。

## 4. データ項目（想定）
フーディスト1名に対して管理する主なデータ項目：
* **基本情報**（氏名、活動名、メールアドレス、拠点エリア）
* **属性・資格**（管理栄養士、フードコーディネーター、調理師等）
* **SNS情報**（Instagram, X, YouTube等のURLおよび最新のフォロワー数）
* **得意分野・対応可能業務**（レシピ開発、写真撮影、動画制作、イベント登壇など）
* **過去の実績・ポートフォリオ画像**
* **社内評価・特記事項**（社内メンバーのみ閲覧可能なメモ、NG事項など）
